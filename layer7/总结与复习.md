# Layer 7 总结与复习

Layer 7 到这里就走完了。这一层只回答了一件事：**协作中的两个 agent，怎么把钱自主、可审计、安全地从一方转到另一方。** 前面的信任、发现、协作都是铺垫，这一层是"agent 经济"真正能跑起来的那一层。

## 一句话贯穿全层

```text
x402 = 钱【真的付了】（HTTP 里带着 EIP-3009 授权，一次往返把稳定币结清）
AP2  = 这笔钱【被批准了】（高风险支付先拿一张签过的 mandate，只授权、不动钱）

  一次 HTTP 往返完成支付：client 请求 → server 回 402 报价 → client 签授权带 proof 重试
  facilitator【验证再放行】：金额/链/币/收款方/时间窗/防重放全对上，才结算、才放资源
  支付是【可审计的状态机】：quote_pending→…→settled，卡在哪、为什么、能不能重试/退款都答得出
  授权与结算必须【绑定同一 task / 金额 / 币 / 链 / 收款方】：任一不符就拒绝
```

把这四行背下来，Layer 7 的骨架就在了：**x402 结算 + AP2 授权 + 一次往返 + 验证再放行 + 状态机可审计 + 绑定一致**。

## 分模块回顾（7 条）

1. **模块 1 · 为什么 agent 需要自主支付**：agent 付款 ≠ 转一次账。它要能自己发现该付多少、自己签授权、自己带 proof 重试，还要能被审计、能被授权闸门卡住。分工是 **AP2 管"谁批准"、x402 管"钱真付了"**。
2. **模块 2 · x402 协议与 HTTP 402 握手**：`client 请求 → server 回 402 + payment-required(b64 的 PaymentRequirements) → client 选一个 requirement、签 EIP-3009 授权、带 proof 重试 → facilitator 验证并结算 → server 放行`。`PaymentRequirements` 讲清"怎么付"（scheme/network(CAIP-2)/amount/asset/payTo/maxTimeoutSeconds）。
3. **模块 3 · EIP-3009 与稳定币 gasless 结算**：`transferWithAuthorization` 让 USDC 免 gas 授权转账，是 x402 的结算底层。USDC 6 位小数（`1000` = `$0.001`），Base Sepolia USDC = `0x036CbD53842c5426634e7929541eC2318f3dCF7e`，facilitator 负责把授权验证并上链结算。
4. **模块 4（核心）· 用 x402 SDK 做付费服务**：server 侧 `paymentMiddleware` + `x402ResourceServer` + `ExactEvmScheme` + `HTTPFacilitatorClient` 把 `/premium` 变成 `$0.001` 付费资源；client 侧 `wrapFetchWithPayment` + `x402Client` 自动付款。不手写 402 握手、proof 解析、签名重试。
5. **模块 5 · 支付状态机与对账**：支付不是"付没付"这个布尔量，是一串**可审计状态**：`quote_pending → awaiting_mandate → authorized → submitted → settled`，任一步可 `failed / expired`，`settled` 后可 `refunded`；非法跃迁必须被拒。对账簿把 `task ↔ mandate ↔ tx ↔ 金额` 串起来，重复回执要**幂等**，迷失/迟到回执要能识别。
6. **模块 6（轻量）· AP2 授权层**：三种 mandate（**Intent / Cart / Payment**）是 W3C VC 风格、ECDSA over P-256 + SHA-256 签名。AP2 只授权不动钱、绑定到用户不绑定 agent；高风险金额先拿一张签过的 mandate 才准结算。`a2a-x402` extension 把 mandate 接到 x402 结算。
7. **模块 7 · 里程碑 Paywalled Agent Service**：把上面全部串成一个闭环——`报价 → (高风险)授权 → 结算 → 回执 → 对账`，并接回北极星业务：A 通过 Layer 6 发现 B、通过 Layer 7 为 B 的服务付 USDC，高风险走 AP2 人工审批，全链路可审计。

## 概念对照表（一行一个）

| 概念 | 一句话含义 |
| --- | --- |
| **x402** | 让"付费"变成 HTTP 状态码：请求付费资源先收 `402`，带上付款 proof 再重试即可拿到内容 |
| **AP2** | 授权层：把"这笔钱被谁批准了"写成签名的 mandate，只授权不动钱，绑定用户 |
| **402 握手** | `请求 → 402 报价 → 签授权带 proof 重试 → facilitator 结算 → 放行`，一次往返完成支付 |
| **PaymentRequirements** | server 在 402 里下发的"怎么付"：scheme / network(CAIP-2) / amount / asset / payTo / 时间窗 |
| **EIP-3009** | `transferWithAuthorization`，稳定币 gasless 授权转账，是 x402 的结算底层 |
| **facilitator** | 第三方验证并上链结算的服务：验金额/链/币/收款方/时间窗/防重放，再放行 |
| **mandate** | AP2 的签名授权凭证，分 Intent / Cart / Payment 三种，ECDSA P-256 签 |
| **状态机** | 支付的可审计生命周期：`quote_pending→…→settled`，非法跃迁被拒，能答"卡在哪" |
| **对账** | 把 `task ↔ mandate ↔ tx ↔ 金额` 串成一条可查记录，重复回执幂等、迷失回执可识别 |
| **wallet session** | 认证一次后拿 session token、按周期结算，为高频调用省掉每笔重签的开销 |

## 可运行产物清单（以实测为准）

两套脚手架都已【实测跑通】。`payment-lab` 本地不联网、不花钱、无私钥；`paywalled-service` 真连公共 facilitator + Base Sepolia USDC。

**payment-lab/**（`cd layer7/payment-lab && pnpm install`）：

```bash
pnpm demo:handshake   # 01 402 握手：server 报价 → client 拼 payment
pnpm demo:verify      # 02 facilitator 校验：金额/链/币/收款方/时间窗/防重放
pnpm demo:state       # 03 支付状态机：合法路径 ✅ / 非法跃迁 ⛔
pnpm demo:recon       # 04 对账：task↔mandate↔tx↔金额 串起来 + 重复回执幂等
pnpm demo:mandate     # 05 AP2 mandate：P-256 签/验 + 高风险阈值审批
pnpm demo:interop     # 06 拒绝矩阵：10 条自断言（伪造 proof/金额不足/错收款方/过期/重复/挪用）
pnpm demo:all         # 依次跑 01–06
```

预期：`demo:all` 全绿（`✅通过 / ⛔拒绝` 逐条打印）；`demo:interop` 的 **10 条自断言 10/10 过**（每条断言"实际 == 预期"，一条不符即红）。

**paywalled-service/**（`cd layer7/paywalled-service && pnpm install && cp .env.example .env`）：

```bash
pnpm start:server     # 进程①：@x402/express 把 /premium 设为 $0.001 付费资源
pnpm inspect          # 进程③【无需私钥】：裸 fetch 打印真实 402 + payment-required header
pnpm start:client     # 进程②【需测试钱包】：wrapFetchWithPayment 自动 402→签名→结算→拿内容
pnpm typecheck        # tsc --noEmit
```

预期：
- `pnpm inspect` **无需私钥**，实测真实返回 `402 Payment Required` + `payment-required` header（b64 解出是 `{"x402Version":2,...}` 的 `PaymentRequirements`）——这是握手第一步，人人都能验。
- `pnpm typecheck` 通过。
- `pnpm start:client` 的真实付款结算半段**需要你自己填 `.env`**（`PAY_TO` + `CLIENT_PRIVATE_KEY`）并领测网 USDC（<https://faucet.circle.com>，选 Base Sepolia），和 Layer 4 的真实链脚本一致——脚手架实测到"server 真实返回 402"，付款那半段读者自己跑。

## 诚实边界（务必记住）

- **x402 用 V2 的 scoped 包 `@x402/*`（实测 v2.17.0）**：`@x402/core`、`@x402/evm`、`@x402/express`、`@x402/fetch`——**不是**旧的扁平 `x402-express` v1.x。server 侧 `paymentMiddleware` + `x402ResourceServer` + `ExactEvmScheme`(来自 `@x402/evm/exact/server`) + `HTTPFacilitatorClient`(来自 `@x402/core/server`)；client 侧 `wrapFetchWithPayment` + `x402Client`(来自 `@x402/fetch`) + `ExactEvmScheme`(来自 `@x402/evm/exact/client`) + `toClientEvmSigner`(来自 `@x402/evm` 根)。`network` 用 CAIP-2。动手前以当前 npm 包为准。
- **`payment-lab` 是本地原理演示，不是真实 SDK**：`PaymentRequirements` / EIP-3009 授权 / AP2 mandate 都是本地类型 + Node 内置 `crypto` 手写（AP2 用 P-256 对应真实 ECDSA），不签真交易、不联网。真实端到端在 `paywalled-service`。
- **真付款需要读者自备测试钱包**：填 `.env` 的测网私钥 + 领 Base Sepolia USDC。私钥**只进 `.env`**（`.gitignore` 排除），不进前端/日志/仓库，只用测试网私钥。
- **AP2 本课轻量处理**：本地 mandate 演示，不接真实 AP2 SDK；治理上 x402 已于 2026-04 归 Linux Foundation 下的 x402 Foundation（Google/Visa/Stripe/Mastercard/Circle 等 20+ 成员）。

## 下一步接 Layer 8 / Capstone

到这里，agent 已经能"自主付款/收款"了。但"能付钱"不等于"付钱安全"——真正上生产前，还差一层把攻击面写成测试、写进 CI 的硬功夫。

```text
Layer 7（本层）：x402 结算 + AP2 授权（agent 能自主、可审计地付钱）
Layer 8       ：安全加固 —— 把"支付重复提交 / 伪造 proof / 过期 quote / 金额不足 / 错收款方"
                这套拒绝矩阵从 demo 里的自断言，升级成进 CI 的回归测试；session key 限额、
                mandate 绑定、facilitator 信任边界都要有测试兜底
Capstone      ：把 A2A 发现 + ERC-8004 信誉 + x402 支付串成完整 agent 服务市场 ——
                A 发现 B、看 B 的信誉、为 B 的服务付 USDC、高风险走 AP2 审批，全链路可审计
```

一句话收尾：**Layer 6 让 agent 能互相发现、协作、交接任务；Layer 7 让协作中的 agent 能自主、可审计、安全地转移价值。** 有了这两层，"agent 经济"才有了发现和结算这两条腿。

## 本模块小结

- 全层一句话：**x402 = 钱真付了；AP2 = 这笔钱被批准了；一次 HTTP 往返完成支付；facilitator 验证再放行；支付是可审计状态机；授权结算必须绑定同一 task/金额/币/链/收款方。**
- 7 个模块串起来：为什么要付 → 402 握手 → EIP-3009 结算底层 → 用 SDK 起真付费服务 → 状态机与对账 → AP2 授权 → 里程碑闭环。
- 概念对照表覆盖 x402 / AP2 / 402 握手 / PaymentRequirements / EIP-3009 / facilitator / mandate / 状态机 / 对账 / wallet session 各一行。
- 两套脚手架实测跑通：`payment-lab` 六个 demo 全绿（`demo:interop` 10/10 自断言过）；`paywalled-service` 的 `pnpm inspect` 无需私钥即返回真实 402，真付款半段读者填 `.env` 自己跑。
- 边界诚实标注：x402 是 v2 scoped 包不是 v1；`payment-lab` 是本地原理演示非真实 SDK；AP2 本地演示非真实 SDK；真付款需读者自备测试钱包。
- 下一步：Layer 8 把拒绝矩阵升级成进 CI 的安全回归；Capstone 把发现 + 信誉 + 支付串成完整 agent 服务市场。
