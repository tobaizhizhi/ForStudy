# 模块 4（核心）：用 x402 SDK 做付费服务

这是 Layer 7 的心脏。前面三模块讲清了 x402 的握手（模块 2）和 EIP-3009 结算底层（模块 3），现在动手：**用官方 SDK 把一个普通 API 变成付费资源，让 agent 自动付款调用它。**

> 目标：用 `@x402/*` v2 的中间件起一个真实付费服务，client 一行 `wrapFetchWithPayment` 就能自动完成"402 → 签名 → 重试 → 拿到结果"。不手写 402 握手、proof 解析、结算撮合。

## 5.1 先讲清楚：为什么不手写 402

模块 2 你已经看到 402 握手的全过程了。你可能会想："流程我懂了，自己按流程写一遍不就行？"——**不要**。手写 402 支付意味着你要自己：

- 拼 `PaymentRequirements`、正确编码进响应头；
- 解析 client 带来的 `payment-signature`、验 EIP-712 签名；
- 调 facilitator 的验证/结算接口、处理它的各种错误码；
- 防重放、防金额篡改、防错收款方……

这些每一条错了都是**真实资金损失**。这正是本路线图"成熟工具优先"原则最该应用的地方：x402 有 production-ready 的官方 SDK 和框架中间件，把上面这些全包了。你只负责声明"哪个路由、收多少、收到哪"。

## 5.2 类比：付费墙中间件

把 x402 中间件想成 **Express 的一个"付费墙"中间件**：

> 就像 `app.use(authMiddleware)` 会拦下没登录的请求一样，`app.use(paymentMiddleware(...))` 会拦下没付款的请求——自动回 `402 Payment Required` + 付款要求。付了款（且验证通过）的请求才被放行到你的业务 handler。你的业务代码几乎不用改。

## 5.3 版本核验（重要，先看这个）

x402 有**新旧两代包**，别装错：

| | 旧一代 | 新一代（本课用） |
| --- | --- | --- |
| 包名 | `x402-express`、`x402-next`（扁平） | **`@x402/*` scoped 包** |
| 实测版本 | v1.2.x | **v2.17.0** |
| network 写法 | `"base-sepolia"` | **CAIP-2：`"eip155:84532"`** |
| server 拼法 | `paymentMiddleware(payTo, routes)` | **builder：`x402ResourceServer` + `register` + `paymentMiddleware(routes, server)`** |

本课用**新一代 `@x402/*` v2.17**（对应 x402 V2，2025-12 发布：模块化拆包 `@x402/evm`·`@x402/svm`、多链、wallet sessions）。装这几个：

```bash
pnpm add @x402/core @x402/evm @x402/express @x402/fetch express viem
```

> ⚠️ 动手前以当前 npm 包为准——x402 迭代很快，包结构/导出仍可能变。本节代码截至 **2026-07** 实测跑通（`@x402/*` v2.17.0）。

## 5.4 server 侧：把 /premium 变成付费资源

看脚手架 `paywalled-service/src/server.ts` 里**实测跑通**的代码：

```ts
import express from "express";
import { paymentMiddleware, x402ResourceServer } from "@x402/express";
import { ExactEvmScheme } from "@x402/evm/exact/server";
import { HTTPFacilitatorClient } from "@x402/core/server";
import { FACILITATOR_URL, NETWORK, PAY_TO, PORT, PRICE } from "./config.js";

const app = express();

// facilitator 负责验证付款、上链结算；用公共测试 facilitator，不自建。
const facilitator = new HTTPFacilitatorClient({ url: FACILITATOR_URL });

// resource server：为 Base Sepolia 注册 exact-evm 结算方案。
const resourceServer = new x402ResourceServer(facilitator)
  .register(NETWORK, new ExactEvmScheme()); // NETWORK = "eip155:84532"

// 把 /premium 挂上付费闸门：每次 GET 收 PRICE 的 USDC 到 PAY_TO。
app.use(
  paymentMiddleware(
    {
      "GET /premium": {
        accepts: {
          scheme: "exact",
          price: PRICE,          // "$0.001"
          network: NETWORK,      // "eip155:84532"（CAIP-2）
          payTo: PAY_TO,         // 你的测试收款钱包
        },
        description: "一条付费的链上数据分析结果（教学示例）",
      },
    },
    resourceServer,
  ),
);

// 付费通过后才会走到这里，返回真正的资源。
app.get("/premium", (_req, res) => {
  res.json({ result: "这是付费内容：agent B 的链上分析结果" });
});

app.listen(PORT, "0.0.0.0", () => {
  console.log(`[server] 付费资源：GET /premium  价格：${PRICE}  收款：${PAY_TO}`);
});
```

三个关键点：

1. **`x402ResourceServer(facilitator).register(network, scheme)`** —— builder 模式，为某条链注册某个结算方案。`ExactEvmScheme` 是"固定价"方案（付固定金额换资源）。
2. **`paymentMiddleware(routes, resourceServer)`** —— 第一参是路由配置，第二参是 resource server。路由 key 是 `"GET /premium"` 这种 `方法 + 路径`。
3. **`accepts`** 里声明 `scheme` / `price` / `network` / `payTo` —— server 就靠这个生成 402 响应里的付款要求。

## 5.5 先看握手：inspect（不需要私钥、不花钱）

付款要真钱，但**看 402 握手不需要**。脚手架 `src/inspect-402.ts` 用裸 `fetch` 请求付费资源，打印 server 回的 402：

```ts
const res = await fetch(`${SERVICE_URL}/premium`); // 不带任何付款
console.log(`HTTP 状态：${res.status} ${res.statusText}`);
if (res.status === 402) {
  console.log("✅ 如预期返回 402 Payment Required");
  console.log("payment-required:", res.headers.get("payment-required"));
}
```

起 server + 跑 inspect：

```bash
pnpm start:server    # 终端 1
pnpm inspect         # 终端 2
```

实测输出：

```text
HTTP 状态：402 Payment Required
✅ 如预期返回 402 Payment Required —— 这就是 x402 握手的第一步。
相关响应头：
  payment-required: eyJ4NDAyVmVyc2lvbiI6MiwiZXJyb3IiOiJQYXltZW50...
```

那个 `payment-required` 头是 base64 编码的 `PaymentRequirements`，解出来就是 `{"x402Version":2, ...}`——**server 端拼法正确、402 握手真实工作**，这一步无需任何钱包即可验证。

## 5.6 client 侧：wrapFetchWithPayment 自动付款

client 侧更省事——包一层 `fetch`，遇到 402 自动签 EIP-3009 授权、带 proof 重试。看 `src/client.ts`：

```ts
import { createPublicClient, http } from "viem";
import { privateKeyToAccount } from "viem/accounts";
import { baseSepolia } from "viem/chains";
import { wrapFetchWithPayment, x402Client } from "@x402/fetch";
import { ExactEvmScheme } from "@x402/evm/exact/client";
import { toClientEvmSigner } from "@x402/evm";
import { CLIENT_PRIVATE_KEY, NETWORK, RPC_URL, SERVICE_URL } from "./config.js";

const account = privateKeyToAccount(CLIENT_PRIVATE_KEY as `0x${string}`);
const publicClient = createPublicClient({ chain: baseSepolia, transport: http(RPC_URL) });

// 组装 x402 client：为 Base Sepolia 注册 exact-evm 客户端方案（用我们的签名器）。
const signer = toClientEvmSigner(account, publicClient);
const client = new x402Client().register(NETWORK, new ExactEvmScheme(signer));

// 包一层 fetch：遇到 402 自动付款重试。
const fetchWithPay = wrapFetchWithPayment(fetch, client);

const res = await fetchWithPay(`${SERVICE_URL}/premium`);
console.log(`最终 HTTP 状态：${res.status}`); // 付款成功后是 200
console.log("拿到付费内容：", await res.json());
```

注意导入路径的细节（都实测核对过）：

- server 侧的 `ExactEvmScheme` 从 **`@x402/evm/exact/server`**；
- client 侧的 `ExactEvmScheme` 从 **`@x402/evm/exact/client`**；
- `toClientEvmSigner` 从 **`@x402/evm`**（根，不是 `exact/client` 子路径）。

这类子路径差异，正是"照着当前 SDK 版本写、别凭记忆"的原因。

## 5.7 端到端：一次自动付费调用

```text
client fetchWithPay("/premium")
  → server 回 402 + payment-required（PaymentRequirements）
  → wrapFetchWithPayment 自动：选 requirement、用 signer 签 EIP-3009 USDC 授权
  → 带 payment-signature 重试
  → server 交给 facilitator 验证 + 上链结算 USDC（gasless）
  → 验证通过，server 放行 /premium 的业务 handler
  → client 拿到 200 + 付费内容
```

真跑（需要测试钱包 + 测网 USDC）：

```bash
cp .env.example .env
# 在 .env 填 PAY_TO（收款）和 CLIENT_PRIVATE_KEY（付款钱包）
# 领测网 USDC：https://faucet.circle.com（选 Base Sepolia）
pnpm start:server    # 终端 1
pnpm start:client    # 终端 2
```

## 5.8 诚实边界：脚手架验证到哪一步

和 Layer 4 的真实链脚本一样，这里也有明确的验证边界：

- **无需私钥即可验证的半段**：`pnpm inspect` 实测 server 真实返回 `402 Payment Required` + `payment-required` header——这证明 server 中间件拼法正确、握手真实工作。
- **需要读者自己跑的半段**：签名付款 + facilitator 结算，需要一个有 Base Sepolia 测网 USDC 的测试钱包。脚手架代码结构完整、API 正确，你填好 `.env` 就能跑通真实付款。

写作环境没有测试钱包，所以本课把可验证的半段实测给你看，付款半段留给你动手——这不是偷懒，是把"哪一步靠什么保证"讲清楚。

## 本模块小结

- 用 `@x402/*` v2 官方中间件做付费服务，不手写 402 握手 / proof 解析 / 结算撮合。
- server 侧：`x402ResourceServer(facilitator).register(network, new ExactEvmScheme())` + `paymentMiddleware(routes, server)`，路由 `accepts` 声明 scheme/price/network(CAIP-2)/payTo。
- client 侧：`wrapFetchWithPayment(fetch, x402Client().register(...))` 一行搞定自动付款。
- `inspect`（裸 fetch）无需私钥即可验证 402 握手；真实付款需测试钱包 + 测网 USDC。
- 认准 `@x402/*` v2 scoped 包（v2.17），不是旧的 `x402-express` v1.x；network 用 CAIP-2。
